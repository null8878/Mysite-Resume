Collaboration

Working With Me

Six things that show up in pull requests, design docs, and on-call rotations. Not aspirational — just operational defaults.

01

Async-first, written

I prefer Slack threads over meetings when the problem fits in prose. Decisions in writing survive the “what did we decide last week?” failure mode.

02

PR review in 24h, batched

Small diffs preferred. Comments arrive in one pass so you don’t fix one nit across five review rounds.

03

Reads the failing log first

Before asking for repro. If I can’t see the failure, I haven’t started. “Works on my machine” is not a debugging step.

04

Tests before “done”

Local manual for the path. Automated for the contract. Integration test for any API change — because that’s where the user feels it.

05

ADRs for non-trivial work

One Markdown file per architecture choice, in the repo, dated. The decision, the alternatives, and why the others would have been wrong for us.

06

Documents trade-offs

Not just “what we picked.” Also “what we didn’t and why.” The avoid-list is what protects future-you from re-debating settled decisions.
